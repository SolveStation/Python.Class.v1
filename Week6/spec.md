# Week 6 – Authentication & Authorization (JWT)

## Learning Objectives
- Understand JSON Web Tokens (JWT) and their role in stateless authentication.
- Implement secure password storage and verification.
- Build registration and login endpoints that issue and validate JWTs.
- Protect API routes using token‑based authorization.
- Apply role‑based access control (RBAC) for different user types.
- Follow best practices for token expiration, refresh, and revocation.

## Project Overview
Create a lightweight authentication service that supports:
1. **User Registration** – `POST /api/auth/register` (store username, email, hashed password).
2. **User Login** – `POST /api/auth/login` (validate credentials, return JWT access token).
3. **Protected Notes API** – reuse the Week 4 note‑taking endpoints but require a valid JWT.
4. **Role Guard** – optional `admin` role that can perform extra actions.

The service will be built with **FastAPI** (async‑first, modern) and store user data in memory (later weeks will persist to PostgreSQL).

## Core Features
| Feature | Description |
|---------|-------------|
| **Password Hashing** | Use `passlib` with bcrypt to hash passwords before storage. |
| **JWT Creation** | Generate signed access tokens (`python-jose`) with configurable expiration. |
| **Token Validation** | Implement `get_current_user` dependency that extracts and validates the token. |
| **Registration & Login** | Validate input, create user records, and return JWT on successful login. |
| **Protected Routes** | Apply `require_token` dependency to secure note‑taking endpoints. |
| **RBAC (Optional)** | Include a `role` field; demonstrate admin‑only endpoints. |
| **Documentation** | Automatic OpenAPI docs generated by FastAPI (Swagger UI). |

## Quick Start
1. **Create a virtual environment**  
   ```bash
   python -m venv venv
   source venv/bin/activate   # Windows: venv\Scripts\activate
   ```

2. **Install dependencies**  
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the server**  
   ```bash
   uvicorn main:app --reload
   ```

4. **Test the API** (using curl, HTTPie, or the automatically generated Swagger UI at `http://127.0.0.1:8000/docs`):
   - `POST /api/auth/register` with JSON `{ "username": "...", "email": "...", "password": "..." }`
   - `POST /api/auth/login` with `{ "username": "...", "password": "..." }` → receive `access_token`.
   - Use `Authorization: Bearer <token>` in subsequent requests to `/api/notes/...`.
